This project is a comprehensive task management web application built with Spring Boot, designed to provide users with a secure and efficient way to manage their personal tasks. Users can register accounts, authenticate securely, and perform complete CRUD (Create, Read, Update, Delete) operations on tasks via an intuitive, responsive web interface powered by Thymeleaf templates and Bootstrap styling.

Key features include robust user authentication with BCrypt-encrypted passwords, session-based security through Spring Security, and data persistence using SQLite database integrated with JPA/Hibernate. The application adheres to MVC architecture, separating concerns into controllers for handling web requests, services for business logic, repositories for data access, and models for entity representation.

The technology stack comprises Java 17, Spring Boot 3.x, Maven for build management, and includes REST API endpoints for programmatic access to task data. The project demonstrates best practices in Spring Boot development, such as layered architecture, CSRF protection, and automatic database initialization on startup.

Project structure organizes code into logical packages: controllers for web endpoints, models for JPA entities (User and Task), repositories for data operations, services for logic, and configuration classes for security and database setup. It supports both web-based interaction and JSON API consumption, making it versatile for various use cases.

To get started, users need Java 17+ and Maven 3.6+. The application can be run with 'mvn spring-boot:run' after cloning and navigating to the project directory. It auto-initializes the database and creates a test user (email: test@example.com, password: Test1234). Access via http://localhost:8080.

API endpoints include authentication routes like GET /sign-in, POST /login, GET /sign-up, POST /sign-up, and POST /logout. Task endpoints feature GET /tasks for viewing tasks, POST /tasks for creation, and RESTful JSON APIs such as GET /api/tasks, POST /api/tasks, PUT /api/tasks/{id}, DELETE /api/tasks/{id}.

The database schema consists of a users table with id, username, password, full_name, email, and a tasks table with id, title, user_id (foreign key). Security implementation uses BCrypt for password encryption, session management, CSRF protection, and user data isolation.

Development notes highlight auto-initialized database, test user creation, logging for debugging, and hot reload for templates. The project encourages following Java coding standards, adding tests for new features, updating documentation, and ensuring security best practices.

Overall, this application serves as an excellent example of modern web development with Spring Boot, emphasizing security, scalability, and clean code principles. It is developed for educational and demonstration purposes, showcasing integration of various Spring ecosystem components.
